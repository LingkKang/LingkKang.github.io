Advanced Concatenate with Sqlite3

CONCATENATE() is a useful function which can be found in Excel, numpy, etc. It could be simply interpreted as adding two or more strings together. The concatenate operator || could be used to add strings, but only use || can not satisfy us in much complicated context, i.e., string separated across columns. But as a powerful database manage system, Sqlite3 (and probably other DBMS) allows us to do this advanced CONCATENATE, even though there is no keyword or function written as CONCATENATE in SQL.

Requirement

Print a single string containing all the letters in ascending order, and letters are separated by one comma and one space.

Implement

So by means of CONCATENATE, we want to output some stuff like A, B, C, D, E, F, G, H, I, J. Since Letters are stored in columns, it dosen’t work if we only use concatenate operater ||.

The requirement sounds difficult, but if you see codes above, there’s nothing too misterious actually.

Firstly, we need to create a helper table CTE, which stands for Common Table Expression. The first row of CTE is the first row of Alphabet, this is where we begin our story.

Next step is simple, we just need to add new lines into it, for each line, it’s Letter is the concat of previous line (in CTE) and previous letter (in Alphabet).

In line 17, it seems like we JOIN the CTE to itself, and here is explaination. We actually JOIN the newly generated line to the previous line, and you don’t need to worry ID exceeds boundary. Don’t confuse it with Self-Join. It is another topic and out of scope of this blog.

Note that in line 18, it’s Alphabet.ID = CTE.ID + 1, and this + 1 is very important. If missing it, the query goes infinitly, and actually generate something like A, A, A, A, A, A, ... endlessly.

And we use UNION to combine two selected table.

Remark
